1. EXECUTIVE SUMMARY

NEXUS AI is a next-generation, dark-mode AI news aggregation platform powered by a high-performance scraping engine that continuously harvests the latest developments across the artificial intelligence ecosystem. The platform aggregates AI news, product launches, LLM model releases, startup funding rounds, and monetization stories from across the internet every 2 minutes — delivering a Bloomberg Terminal-grade experience for the AI community.

To become the definitive real-time intelligence hub for AI professionals, developers, investors, and enthusiasts — the first screen they check every morning and the last before bed.

4. ADVANCED SCRAPING ENGINE

The scraping engine is the core competitive moat of NEXUS AI. It is a distributed, fault-tolerant, multi-source ingestion pipeline capable of processing hundreds of sources simultaneously with a 2-minute refresh cycle.

4.1 Architecture Overview
FLOW
Source Registry → Scheduler → Scraper Workers (Kafka) → Content Extractor → Deduplicator → Enrichment Engine → MongoDB → Elasticsearch Index → WebSocket Broadcast

4.3 Scraping Worker Design
4.3.1 Worker Pool Architecture
	•	Spring @Scheduled triggers Quartz jobs every 2 minutes per source group
	•	Each source assigned to a Kafka topic partition — parallel processing
	•	Playwright-based headless browser workers for JavaScript-rendered pages (SPA sites)
	•	Lightweight Jsoup workers for static HTML sources (faster, lower resource)
	•	Auto-detection: if Jsoup fetch returns minimal content, escalate to Playwright

4.3.2 Intelligent Rate Limiting & Anti-Detection
	•	Per-domain rate limiting with exponential backoff on 429/503 responses
	•	Rotating residential proxy pool integration (BrightData / Oxylabs)
	•	User-agent rotation from a curated pool of 200+ real browser signatures
	•	Random delay jitter (500ms–3000ms) between requests per domain
	•	Cookie jar management per domain session
	•	CAPTCHA detection with automatic proxy rotation on trigger

4.3.3 Content Extraction Pipeline
	•	Mozilla Readability (via GraalVM JS) for article body extraction
	•	OpenGraph / Schema.org metadata parsing for title, image, author, published date
	•	Named Entity Recognition (NER) using Stanford CoreNLP to tag companies, people, products
	•	Keyword extraction using TF-IDF against AI domain vocabulary
	•	Language detection — English-only by default with multilingual flag support
	•	Toxicity/spam scoring to filter low-quality content

4.3.4 Deduplication Engine
	•	SHA-256 fingerprint on normalized URL (strip UTM params, lowercase)
	•	SimHash-based near-duplicate detection for articles with different URLs but identical content (>85% similarity threshold)
	•	Redis bloom filter for O(1) URL seen-check before heavy processing
	•	Cross-source deduplication: same story from TechCrunch + Wired merged with canonical pointing to original source

4.4 AI Enrichment Layer
	•	Category tagging: Model Release, Funding, Open Source, Research, Product Launch, Monetization Story, Regulation
	•	Company tagging: Automatic extraction and normalization to canonical company names
	•	Sentiment scoring: Bullish / Neutral / Bearish for startup/market stories
	•	Impact score (0–100): Algorithmic scoring based on source authority, social signals, novelty
	•	Summary generation: GPT-4o / Claude Haiku API for 2-sentence TLDR per article
	•	Embedding generation: OpenAI text-embedding-3-small for semantic search and related articles

4.5 Scrape Health & Monitoring
	•	Grafana dashboard per source: success rate, last success timestamp, avg latency, articles/hour
	•	PagerDuty alerts when source fails for > 10 minutes
	•	Dead letter queue (Kafka DLQ) for failed scrape jobs with retry policy
	•	Admin panel: manually trigger re-scrape, pause/resume sources, view raw HTML

5.3 Trending Algorithm
Formula
TrendScore = (likes × 3 + views × 1 + saves × 5 + shares × 4) × DecayFactor(publishedAt) × SourceAuthority × NoveltBonus
The decay factor applies exponential time decay with a 6-hour half-life, ensuring fresh content always surfaces. The novelty bonus rewards articles about genuinely new topics not seen in the past 48 hours, calculated using embedding cosine similarity against recent corpus.

10. RISKS & MITIGATIONS

Risk
Mitigation
Source blocks scraping
Residential proxy rotation, rate limiting, Playwright headless browser for SPAs
LLM API cost overrun
Cache all TLDRs; batch embeddings; use Haiku/GPT-3.5 for summaries not GPT-4
Near-duplicate flood
SimHash dedup + bloom filter prevents 90%+ duplicates before DB write
Elasticsearch index lag
Async indexing via Kafka; search uses eventually-consistent model with < 5s delay
MongoDB write throughput
Time-series collection for interactions; write batching; Atlas auto-scaling
WebSocket scalability
Spring + STOMP over Redis pub/sub — horizontally scalable notification fanout
